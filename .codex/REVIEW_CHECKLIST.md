# REVIEW CHECKLIST

交付前请按以下清单逐项确认：

## A. 需求与范围

- 任务目标已完成，未偏离用户要求。
- 仅修改约定范围内文件，无无关改动。
- 若有假设，已在交付说明中写清。

## B. 代码与模型

- 模型目标函数、约束、变量类型与问题定义一致。
- `problem_type` 与模型类别匹配。
- 公共逻辑优先沉淀到公共模块（如 `src/ModelSolver.py`）。
- 兼容入口脚本保持薄封装，不重复核心实现。

## C. 文档与可读性

- README 与实际脚本、目录一致。
- 数学模型说明包含变量定义、目标函数、约束公式。
- 注释简洁且有信息量，无占位注释。
- UTF-8 编码无乱码。

## D. 验证与风险

- 已执行语法检查（至少 `py_compile`）。
- 已运行关键脚本并核对关键输出。
- 无法验证的部分已明确标注风险和原因。

## E. 交付与发布

- 交付说明包含：改了什么、验证了什么、剩余风险。
- 按用户要求决定是否提交/推送。
- 仅在用户明确要求 `push code` 时推送远端。
